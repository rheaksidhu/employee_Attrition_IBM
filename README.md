# V1 Baseline: Initial AI-Generated Logistic Regression

This branch contains the unmodified initial code generated by GPT-5 (V1a) and Claude Sonnet 4.6 (V1b) using the final code generation prompt (1f). Both versions crashed before producing output due to a pandas StringDtype compatibility issue. Neither model tested its output against the dataset before returning the code.

## Code Generation Prompt
The prompt used to generate both V1a and V1b is documented in [Meta_Prompt_Development.md](../Meta_Prompt_Development.md) — see prompt **1f**.

## Key Issues Identified
- Both versions crashed due to pandas StringDtype compatibility. LabelEncoder skipped all categorical columns
- 86% accuracy reported but functionally useless. V1b predicted "No Attrition" for every employee (0 true positives), V1a caught only 3. Both models underfit the positive class entirely
- LabelEncoder applied to nominal variables imposing false ordinal relationships
- No feature scaling applied. MonthlyIncome artificially dominant
- No class weighting, model optimises for majority class only
- No cross-validation, single 80/20 split unreliable on imbalanced data
- Protected characteristics (Age, Gender, MaritalStatus) included as training features
- Accuracy used as sole metric, inappropriate for imbalanced data
